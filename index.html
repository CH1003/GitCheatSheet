Git Cheatsheet

Moverse a la carpeta deseada

dir -> Muestra los directorios accesables desde la consola.

cd <Directorio>  -> Entrar al directorio. Ayuda a encontrar el directorio en la computadora para inicializar git.

Commits

git help <comando> -> Ayuda!

git status -> Muestra el estado y cambios a hacer.

git add -A -> Añade todos los archivos nuevos o modificados para luego hacer commit.

git commit -m "<Mensaje>" -> Hace commit con el mensaje.

git commit -am "<Mensaje>" -> Combina "git add -A" y "git commit -m" en un solo comando.

git commit --ammend -m "<Mensaje>" -> Reemplaza el mensaje del commit anterior por si se cometió un error.

git log -> Muestra el historial de commits.

git log > <Nombre>.txt -> Crea un archivo txt con el historial de commits actual. 

git chekout <ID> -> Se devuelve en el tiempo al commit especificado por el identificador.

git chekout master -> Nos devuelve al commit más reciente.

git reset --soft <ID> -> Elimina el commit realizado sin hacer cambios al código.

git reset --hard <ID>-> Elimina el commit y se lleva todo en banda.

Ramas

Sirven para crear alterativas del projecto sin hacer cambios al proyecto master


git branch -> Muestra las ramas.

git branch <Nombre> -> Crea una rama. 

git checkout <Rama> -> Para moverse entre ramas.

git checkout -b <Rama> -> Crea y se mueve a una rama.

git branch -D <Nombre de Rama> -> Elimina la rama.

Fusiones

Absorber en una rama, los cambios de otra (Que una rama de pruebas satisfactorias Test sean el nuevo master, por ejemplo).

Ejemplo
1. Situarse en rama que va a absorber (master).
2. git merge <Rama> (Test).

Tipos de conflictos

	Fast Foward -> Sin problemas.
	Manual Merge -> Problemas que necesitan ser manejados manualmente.

Repositorios remotos

Ejemplos: GitHub, BitBucket, Assembla, etc


git clone <URL> -> Bajamos el repositorio del URL a la carpeta en la que se encuentra la consola.

git remote add origin <Llave SSH del repositorio remoto> -> Conecta el repositorio local con el remoto.

git remote -v -> Muestra el repositorio remoto con el cual se está enlazado.

git remote remove origin -> Elimina la sincronización con el repositorio remoto.

 git push origin <Rama> (master) -> Manda el repositorio local al repo remoto. (Agregar -f para forzar un push)

git tag -a <vX.X> -m "<Mensaje>" *Codigo SHIA del commit a etiquetar*-> Crea una etiqueta para especificar una versión concreta del proyecto.

git push origin vX.X o git push origin --tags -> Para mandar las etiquetas al repositorio.

WorkFlow de Equipo

1.Crear organización en github o bitbucket.
2.Agregar colaboradores.
